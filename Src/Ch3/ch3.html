⚙️ CH03: Functions

Logic Encapsulation & Reusability

Functions allow you to write logic once and execute it anywhere. They are the primary way we organize code into manageable, testable "machines."

1. Function Types (Syntax & Behavior)

How you write a function affects Hoisting (the ability to call a function before it's defined). Declarations are hoisted; Expressions and Arrows are not.

  • Declaration: Defined with the function keyword. Hoisted to the top.
  • Arrow: Compact () => {} syntax. Best for modern, clean code.

Output: Types

Copy & Try (Hoisting vs Arrow):

// 1. Declaration (Hoisted)
console.log(sayHi()); // Works even if called before definition
function sayHi() { return "Hi!"; }

// 2. Arrow (Not Hoisted)
const square = (n) => n * n;
console.log(square(5)); // 25

2. Default Parameters (Error Prevention)

If a user forgets to pass an Argument, the function can use a Default Value. This prevents NaN (Not a Number) errors in math calculations.

  • Missing Args: If empty, the parameter takes the default value.
  • Provided Args: If passed, the default is ignored.

Output: Defaults

Copy & Try (Math Safety):

function calcTotal(price, tax = 0.1) {
  return price + (price * tax);
}
console.log(calcTotal(100)); // 110 (Used default 0.1)
console.log(calcTotal(100, 0.2)); // 120 (Used custom 0.2)

3. Scope & Shadowing (Visibility)

Scope determines where variables live. Shadowing is when a function's inner variable "overrides" a global variable with the same name.

Output: Scope

Copy & Try (Shadowing Trace):

let mode = "Global";
function changeMode() {
  let mode = "Local"; // Shadows global
  return mode;
}
console.log(changeMode()); // "Local"
console.log(mode); // "Global" (Global remains unchanged)

4. Call & Apply (Manual Context)

Usually, this refers to the object that called the function. With .call() and .apply(), you can force this to be whatever object you want.

  • .call(): Pass arguments individually (comma-separated).
  • .apply(): Pass arguments as an Array.

Output: Binding

Copy & Try (Borrowing Methods):

const person = { name: "John Doe" };
function intro(age) { console.log(this.name + " is " + age); }
intro.call(person, 25); // "John Doe is 25"

5. Closures (Encapsulation)

A Closure allows a function to "remember" variables even after the parent function has closed. This is used to create Private Variables that cannot be accessed or changed from the outside.

Output: Closure

Copy & Try (Private Counter):

function createCounter() {
  let count = 0;
  return () => ++count;
}
const myCounter = createCounter();
console.log(myCounter()); // 1
console.log(myCounter()); // 2
