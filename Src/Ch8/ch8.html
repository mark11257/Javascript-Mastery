🌐 CH08: The Browser Object Model (BOM)

Interacting with the Environment

The BOM is a collection of objects that allow JavaScript to "talk" to the web browser. It lives outside the actual HTML document and handles things like window size, URLs, and browser history.

1. The Global Boss: Window

The window object is the top-level object in the BOM. Everything else (including the Document) lives inside it. It represents the actual browser tab or window.

  • Dimensions: innerWidth and innerHeight tell you the size of the viewport.
  • Alerts: Methods like alert() and confirm() are technically part of the window object.

Output: Window

Copy & Try:

console.log("Width: " + window.innerWidth);
window.alert("System check complete.");

2. Navigation: Location & History

The location object acts as your browser's GPS, while history acts as its memory of where you've been.

  • Location: Use it to get the current URL or redirect the user to a new page.
  • History: Allows you to move the user back or forward in their browsing session.

Output: Navigation

Example :

// Check if current URL contains a specific user
if (location.href.includes("JohnDoe")) {
  console.log("Welcome back, John Doe!");
}

3. Browser Identity: Navigator

The navigator object tells you who the user is from a technical perspective—what browser they use, their language, and if they are connected to the internet.

Output: Identity
