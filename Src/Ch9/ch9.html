CH09: The DOM & Persistence

Directing the Digital Stage

JavaScript isn't just for math—it's for Manipulation. We will learn how to rewrite the page on the fly and save user data like John Doe so it survives a page refresh.

1. Traditional Selection

The original methods for finding elements. getElementById returns a single object, while getElementsByTagName returns an HTMLCollection (which acts like an array).

Output: Tradition

Copy & Try:

const myHeader = document.getElementById("firsth");
const allParagraphs = document.getElementsByTagName("p");
console.log("Paragraph Count: " + allParagraphs.length);

2. The Universal Selector

querySelector is the modern standard. It uses CSS syntax. querySelectorAll returns a NodeList, which is a static snapshot of elements.

Output: Query

Copy & Try:

// Finds first element with class 'name'
const subHeader = document.querySelector(".name");
subHeader.style.borderBottom = "1px solid #ff9800";

3. Content Properties

innerHTML parses the string as HTML (dangerous if from users!), while innerText treats it as raw text.

Output: Content

Copy & Try:

const box = document.querySelector(".value3");
box.innerHTML = "<button>I am a real button</button>";
box.innerText = "<button>I am just text</button>";

4. Creating Elements

Elements created with createElement live in memory first. They don't appear on the screen until you append them to a parent node.

Output: Nodes

Copy & Try:

const span = document.createElement("span");
span.innerText = "New dynamic item";
document.body.appendChild(span);

5. Tree Navigation

You can "walk" the DOM using parentElement, nextSibling, and children. This is useful when you don't have IDs for every element.

Output: Family

Copy & Try:

const current = document.querySelector(".chapter-section");
console.log("Parent Tag: " + current.parentElement.tagName);

6. Form Collections

The document.forms collection is an easy way to access all inputs inside a <form> without writing complex selectors.

7. The Cookie Rulebook

A cookie is not just a piece of data. It follows a strict "General Rule" syntax. If you miss a semicolon, the browser won't store it correctly.

Part Example Purpose
Name=Value UserName=JohnDoe The actual data pair.
Expires expires=Sun, 05 Apr 2026... How long the data lives.
Path path=/ Which pages can see the cookie.

The Format Template:

document.cookie = "key=value; expires=date; path=/";
