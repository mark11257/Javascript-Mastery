CH02: The Foundation

Variables, Logic, and Automation

Welcome to the engine room. This chapter covers how JavaScript stores data, makes logical decisions, and repeats tasks so you don't have to.

1. Variables: The Storage Units

Think of a Variable as a labeled box in memory. You put data in, and you give it a name to find it later.

  • const: Use this for "Permanent" data. If you try to change it, the code will crash.
  • let: Use this for data that "Changes" (like a game score or a user's age).
  • var: The "Old Way." It has weird scoping rules that lead to bugs. Don't use it.

Live Output: Variables

Copy & Try this in your console:

const username = "John Doe"; // Locked
let level = 1;
level = 2; // This works!
console.log(username + " is at level " + level);

2. Type Coercion & Equality

JavaScript is "loose." It tries to be helpful by converting types automatically, which can be a nightmare if you aren't careful.

  • Loose Equality (==): Only checks the value. It thinks 5 and "5" are the same.
  • Strict Equality (===): Checks Value + Type. This is what pros use 99% of the time.
  • Math Coercion: Plus (+) usually turns things into Strings (concatenation), while Minus (-) turns them into Numbers.

Live Output: Coercion

Copy & Try:

console.log(10 == "10"); // true (Loose)
console.log(10 === "10"); // false (Strict)
console.log("10" + 5); // "105" (Wait, what?)
console.log("10" - 5); // 5 (Math logic applied)

3. Conditionals: The Fork in the Road

Your code isn't smart until it can choose a path. We use Boolean Logic (True or False) to decide what happens next.

  • If/Else: Great for simple "Yes/No" or "This/That" scenarios.
  • Switch: Cleaner when you have multiple specific cases (like a menu or color picker).

Live Output: Branching

Copy & Try:

let hour = 14;
if (hour < 12) {
  console.log("Good Morning");
} else {
  console.log("Good Afternoon");
}

4. Loops: Stop Repeating Yourself

Loops are for Automation. If you need to do something 100 times, you don't write 100 lines of code; you write one loop.

  • For Loop: Used when you know exactly how many times to run (e.g., "Count to 10").
  • While Loop: Used when you want to keep running until something happens (e.g., "Keep asking for a password until it's correct").

Live Output: Loops

Copy & Try:

for (let i = 1; i <= 5; i++) {
  console.log("Iteration number: " + i);
}
