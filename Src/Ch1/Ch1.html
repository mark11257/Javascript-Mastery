CH01: JavaScript Execution

Interaction & The Console

HTML is just a skeleton. To make it move, we use JavaScript to manipulate the DOM (Document Object Model). If you don't understand how JS hooks into your HTML, your site is basically a digital corpse.

1. Attribute-Level JS (Inline)

This is the "lazy" method. You’re shoving logic directly into HTML Attributes. It’s tightly coupled, meaning if you want to change a function name later, you’ll be hunting through 500 lines of HTML like a madman. It's fine for a quick "Hello World," but don't bring this into a production environment unless you want your team to hate you.

The Implementation:

/* The browser reads the 'onclick' attribute and executes the string as raw JS. Zero separation of concerns. */ <button onclick="alert('Logged')">Execute</button>

2. Decoupled JS (Event Listeners)

This is the industry standard. We keep the Logic in a .js file and the Structure in the .html file. We "grab" an element by its ID and attach an Event Listener. It’s like setting a trap: the code just sits there silently until the user triggers the event (like a click).

System State: IDLE

Step 1: The Target (HTML)

<!-- The ID is the 'hook' that allows JS to find this specific node --> <button id="light-switch">Trigger</button>

Step 2: The Logic (JS)

/* 1. Locate the Object in the DOM tree 2. Attach a Listener for the 'click' event 3. Fire the Callback function when the event happens */ const targetBtn = document.getElementById('light-switch'); targetBtn.addEventListener('click', function() { document.getElementById('room-status').innerText = "ACTIVE"; });

3. Runtime Debugging (The Console)

The console is your MRI machine. If your code is "dead on arrival," this is where you find out why. You can check the Stack Trace, log variables to see where they broke, or inspect the entire Object Tree. If you aren't using the console, you aren't developing; you're just guessing.

Open F12 > Console. If you see red text, you messed up. If it's empty, you're doing okay.

Common Debugging Methods:

console.log("Standard log"); // For general 'I am here' checks console.warn("Soft warning"); // 'This is weird but won't crash... yet' console.error("Hard error"); // Use this when the world is ending console.dir(document); // Dig into the actual Object properties
