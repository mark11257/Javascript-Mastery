🌐 CH11: AJAX & The XHR Object

Mastering the art of Asynchronous communication.

1. What is AJAX?

AJAX stands for Asynchronous JavaScript and XML. It allows you to send and receive data from a server in the background. This means you can update a user's profile picture or load new posts without the user ever seeing a "loading" spinner on the tab.

The "Asynchronous" Part: Your code doesn't stop and wait for the server. It says, "Go get this data, and call me when you're done. I'm going to keep running other things in the meantime."

2. The XMLHttpRequest (XHR) Object

This is the built-in browser object that does the heavy lifting. To use it, you follow 4 steps:

  1. Create the new XMLHttpRequest() object.
  2. Use .open() to set the method (GET/POST) and the URL.
  3. Define the .onreadystatechange or .onload handler.
  4. Call .send().

3. ReadyStates and Status Codes

The XHR object goes through 5 stages of "Readiness." We usually wait for State 4 (Completed).

readyStateMeaning
0 (UNSENT)Client has been created.
1 (OPENED)open() has been called.
3 (LOADING)Downloading; responseText holds partial data.
4 (DONE)The operation is complete!

We also check the HTTP Status. 200 means "OK", while 404 means "Not Found".

4. Processing JSON Data

The server sends data as a **string**. To use it in JavaScript, we must turn that string into an object using JSON.parse().

// Inside the XHR onload function:
let data = JSON.parse(this.responseText);
console.log(data.name); // Accessing your profile.json!
