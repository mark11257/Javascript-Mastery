CH05: Error Handling

Bulletproofing Your Code

Errors are inevitable. Whether it's a network failure or a user typing "banana" into a "number" field, your job is to catch these errors before they crash the entire application.

1. The Error Object & Throwing

JavaScript has a built-in Error class. You can "throw" an error manually to stop execution when a specific condition isn't met.

  • throw: Stops the script and sends a message to the console.
  • Error Types: RangeError (number out of bounds), ReferenceError (variable doesn't exist), TypeError (wrong data type).

Output: Errors

Copy & Try:

if (isNaN(x)) {
  throw new TypeError("Value must be a number!");
}

2. Try, Catch, & Finally

This is the safety net. You try a block of code; if it fails, the catch block handles the mess, and finally runs no matter what (used for cleanup).

Output: Logic

Copy & Try:

try {
  riskyFunction();
} catch (err) {
  console.error("Caught: " + err.message);
} finally {
  console.log("Cleanup complete.");
}

3. Error Categorization (instanceof)

Not all errors are equal. A RangeError (number too big) requires a different response than a ReferenceError (code is broken). We use instanceof to identify the "Species" of the error.

  • ReferenceError: Thrown when you try to use a variable that hasn't been declared.
  • TypeError: Thrown when a value is not the expected type (e.g., calling a string as a function).
  • RangeError: Thrown when a number is outside its allowed range.

Output: Types

Copy & Try (The Dispatcher Pattern):

try {
  // Force a specific error type
  null.f();
} catch (e) {
  if (e instanceof TypeError) console.log("Wrong Type!");
  else if (e instanceof ReferenceError) console.log("Missing Var!");
}

4. Custom Error Classes

For complex apps, the standard JS errors aren't enough. You can create your own Custom Error Classes that inherit from the base Error. This allows you to add custom properties like errorCode or timestamp.

Output: Custom

Copy & Try (Custom Validation Error):

// 1. Define the custom class
class MyError extends Error {
  constructor(message, code) {
    super(message);
    this.name = "DatabaseError";
    this.code = code;
  }
}

// 2. Use it in a try-catch
try {
  throw new MyError("Connection Lost", 503);
} catch (e) {
  console.error(`${e.name} (${e.code}): ${e.message}`);
}
