CH07: Advanced Functions

Lifecycle, Scope, and Context

Functions are more than just blocks of code. They have a lifecycle (Hoisting) and a dynamic relationship with data (The this keyword). Understanding these allows you to write "pluggable" logic.

1. Hoisting & The IIFE Pattern

When JavaScript runs, it looks for function Statements first and moves them to the top of the memory. This is Hoisting. Expressions (anonymous functions assigned to variables) are not hoisted.

  • IIFE: An "Immediately Invoked Function Expression" runs as soon as it is defined. It is used to keep variables private and avoid "polluting" the global scope.
  • Safe Execution: Statements can be called before they appear in the file; Expressions cannot.

Output: Hoisting

Example:

// This works even if called at line 1
function summ() { console.log("Hoisted!"); }

// This will crash if called early
const sum = function() { console.log("Not Hoisted"); };

2. Handling Dynamic Inputs (Arguments)

Sometimes you don't know how many inputs a function will get. JavaScript provides a built-in arguments object—an array-like structure that captures every value passed to the function, regardless of defined parameters.

Output: Arguments

Example:

function logInput() {
  console.log(arguments[0]); // Captures whatever is passed first
}
logInput("John Doe");

3. Method Borrowing (Changing Context)

Functions can be "borrowed" from one object and used on another. By using call, apply, or bind, you manually set what the this keyword refers to.

  • Call & Apply: Execute the function immediately. Apply takes arguments as an array; Call takes them individually.
  • Bind: Does not run the function immediately. It creates a new function with the context locked in for later use.

Output: Borrowing

Example (The Bind Method):

const str = "John Doe";
// Borrow the array 'join' method for a string context
const boundJoin = [].join.bind(str);
console.log(boundJoin(".")); // J.o.h.n. .D.o.e
