⚙️ CH03: JavaScript Functions

Reusable Logic Engines

A function is a block of code designed to perform a specific task. You define it once, and you can invoke it whenever you need to execute that logic.

1. Function Types

How you define a function changes how it behaves in memory (Hoisting) and how it handles the this keyword.

  • Declaration: Standard way. Can be called before it's defined in the code (Hoisted).
  • Expression: Stored in a variable. Not hoisted.
  • Arrow Functions: Shorthand syntax. Doesn't have its own this context.

Live Output Area

Copy & Try:

// Arrow Function (Modern & Clean)
const square = (n) => n * n;
console.log(square(5)); // 25

2. Default Parameters

In older JS, if you forgot an Argument, the parameter became undefined and crashed your math. Now, we set Default Values directly in the signature.

Live Output Area

Copy & Try:

function greet(name = "User") {
  return "Hello " + name;
}
console.log(greet()); // "Hello User"

3. Scope & Shadowing

Scope determines where a variable is accessible. Shadowing happens when a local variable has the same name as a global one—the local one "hides" the global one inside that function.

Live Output Area

Copy & Try:

let val = "Global";
function test() {
  let val = "Local"; // Shadowing
  console.log(val);
}
test(); // "Local"

4. Method Binding

Sometimes you need to borrow a function from one object and use it on another. We use call, apply, and bind to manually set what this refers to.

Live Output Area

Copy & Try:

const dev = { name: "Mark" };
function tellName() { console.log(this.name); }
tellName.call(dev); // Forces 'this' to be 'dev'

5. IIFE & Closures

A Closure is when a function "remembers" its environment even after the outer function has finished executing. An IIFE (Immediately Invoked Function Expression) is a function that runs as soon as it's defined.

Live Output Area

Copy & Try:

(function() { console.log("I run instantly!"); })();
