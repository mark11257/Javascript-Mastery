CH10: The Event Concept

Making the Page Reactive

The Concept: The browser is constantly watching. Every time a user clicks, types, moves the mouse, or loads a page, the browser fires an "Event." JavaScript allows us to write "Handlers" (or Listeners) that wait for these specific events and execute code in response.

1. Properties vs. Listeners & The "this" Keyword

There are two main ways to attach an event:

  • As an Object Property: button.onclick = function() {...}. This is older. You can only have ONE onclick function. If you write another, it overwrites the first.
  • Using addEventListener: button.addEventListener('click', function() {...}). This is modern. You can attach 10 different click functions to the same button and they will all fire.

The "this" Keyword: Inside an event function, this acts as a shortcut referring strictly to the HTML element that fired the event.

button.addEventListener('click', function() {
  // "this" refers to the button itself!
  this.style.backgroundColor = "green";
});

2. The Event Object (e) Properties

When an event happens, JavaScript secretly creates an object packed with details about what just happened and passes it into your function (usually named e or event).

PropertyWhat it tells you
e.targetThe exact HTML element that triggered the event.
e.typeThe name of the event (e.g., "click", "keydown").
e.clientX / YThe X and Y mouse coordinates on the screen.

4. Keyboard Listeners

We attach keydown or keyup usually to the `window`, `document`, or specific input fields. The e.key property tells us exactly what letter or command was struck.

5. Form Events: Focus, Blur & Change

Forms are highly interactive.

  • onfocus: Triggers the moment you click INSIDE an input.
  • onblur: Triggers the moment you click AWAY (lose focus). Great for form validation.
  • onchange: Triggers when you leave the input AND the value was altered.

6. Dropdown (Select) Options

To get data from a dropdown list, we don't just grab its value. A <select> element has an array called options, and a property called selectedIndex.

Live Output: Menu Choice

// How to read a dropdown:
var menu = document.getElementById("myMenu");
var chosenText = menu.options[menu.selectedIndex].text;
var hiddenValue = menu.options[menu.selectedIndex].value;

7. onsubmit & Prevent Default Behavior

The Problem: By default, clicking a submit button inside a form tells the browser to immediately refresh the page and send data to a server.

The Fix: We use e.preventDefault(). This stops the browser's default refresh action dead in its tracks, allowing our JavaScript to step in, validate the data, and send it smoothly behind the scenes.

Live Output: Submission

8. The Event Flow: Bubbling & Capturing

When you click a button inside a div, you are technically clicking BOTH the button and the div. But which event fires first?

  • Bubbling (Default): Events bubble UP from the inside out. (Child triggers first, then Parent).
  • Capturing: Events capture DOWN from the outside in. (Parent triggers first, then Child). We activate this by adding true as the third parameter in addEventListener.

9. Custom Events

You can create your own unique events and pass custom data using the detail property. You "fire" them manually using dispatchEvent.

Live Output: Custom Event

var myEvent = new CustomEvent('seen', { detail: { msg: 'User viewed it!' } });
document.dispatchEvent(myEvent);
