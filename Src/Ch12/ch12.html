CH12: Object-Oriented Programming

Moving from individual variables to Structured Blueprints.

1. The Constructor Function

Before ES6 "Classes," we used functions to build objects. A Constructor is a special function used with the new keyword to stamp out multiple objects of the same type.

Output: Constructor

function User(name, role) {
  this.name = name;
  this.role = role;
}
const user1 = new User("Mark", "Dev");

2. Classes & The Constructor

Modern JS uses the class keyword. It’s cleaner and more readable. The constructor() method inside a class is where you initialize the object's properties.

Output: Classes

3. Encapsulation (Private Data)

Encapsulation is the act of hiding data. In modern JS, we use a hash (#) before a variable name to make it private. It cannot be accessed or changed from outside the class.

Output: Security

class Bank {
  #balance = 0; // Private! Cannot be touched directly.
}

4. Inheritance: The "Extends" Keyword

Inheritance allows one class to derive features from another. A Developer class can inherit from a Person class. We use super() to call the parent's constructor.

Output: Family Tree

5. Polymorphism (Many Shapes)

Polymorphism allows a child class to change how a parent method works. For example, every animal makesSound(), but a Dog barks while a Cat meows.

Output: Sounds
